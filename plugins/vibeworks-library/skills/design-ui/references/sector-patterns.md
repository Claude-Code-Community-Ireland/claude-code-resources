# Sector-Specific Design Patterns

## Fintech / Financial Services

### Visual Language
- **Colors**: Blues, greens for trust; avoid red except for warnings
- **Typography**: Clean sans-serif (Inter, SF Pro, Roboto)
- **Imagery**: Abstract data visualizations, clean illustrations

### Essential Elements
- Security badges and certifications
- Clear data presentation
- Progress indicators for multi-step flows
- Account balance prominence
- Transaction history patterns

### Trust Signals
- SSL/security indicators
- Regulatory compliance badges
- Customer testimonials with real photos
- Clear privacy policy links
- Two-factor authentication indicators

### Common Patterns
- Dashboard with key metrics
- Card-based account displays
- Tabular transaction lists
- Modal confirmations for actions
- Notification systems

---

## Healthcare / Medical

### Visual Language
- **Colors**: Calming blues, greens, whites; careful with red
- **Typography**: Highly readable, larger base sizes
- **Imagery**: Warm, human photography; avoid clinical sterility

### Essential Elements
- Patient information displays
- Appointment scheduling interfaces
- Medical record summaries
- Medication tracking
- Emergency contact prominence

### Accessibility Priority
- WCAG AAA compliance essential
- High contrast modes
- Screen reader optimization
- Large touch targets (min 44px)
- Clear error messages

### Common Patterns
- Timeline views for medical history
- Card-based provider profiles
- Calendar-centric scheduling
- Secure messaging interfaces
- Document upload systems

---

## E-commerce / Retail

### Visual Language
- **Colors**: Brand-driven, with clear CTA colors
- **Typography**: Mix of personality (headings) and clarity (body)
- **Imagery**: High-quality product photography

### Essential Elements
- Product grids with filters
- Shopping cart indicators
- Price prominence
- Add-to-cart buttons
- Size/variant selectors

### Conversion Elements
- Trust badges (payment security)
- Review/rating displays
- Stock indicators
- Shipping information
- Return policy clarity

### Common Patterns
- Product detail pages with galleries
- Faceted search and filtering
- Comparison features
- Wishlist functionality
- Recently viewed products

---

## SaaS / Software

### Visual Language
- **Colors**: Modern, often purple/blue gradients (use thoughtfully)
- **Typography**: Tech-forward but readable
- **Imagery**: Product screenshots, illustrations

### Essential Elements
- Feature showcases
- Pricing tables
- Integration logos
- Testimonials/case studies
- Free trial CTAs

### Onboarding Patterns
- Step-by-step wizards
- Progress indicators
- Empty states with guidance
- Tooltips and hints
- Interactive tutorials

### Common Patterns
- Hero with product screenshot
- Feature grid sections
- Comparison tables
- FAQ accordions
- Footer with navigation

---

## Creative / Agency

### Visual Language
- **Colors**: Bold, unexpected combinations
- **Typography**: Distinctive, personality-driven choices
- **Imagery**: Portfolio work, team culture

### Essential Elements
- Portfolio/work showcase
- Team introductions
- Client logos
- Process explanations
- Contact/inquiry forms

### Personality Expression
- Unique scroll behaviors
- Custom cursors
- Animation and motion
- Unexpected layouts
- Strong brand voice

### Common Patterns
- Case study deep-dives
- Grid-based portfolio
- About page storytelling
- Blog/insights section
- Career pages

---

## General Best Practices

### Above the Fold
- Clear value proposition
- Primary CTA visible
- Navigation accessible
- Brand identity clear

### Mobile Considerations
- Touch-friendly targets (44px min)
- Thumb-zone awareness
- Reduced cognitive load
- Fast loading priorities

### Performance
- Optimized images
- Minimal blocking resources
- Progressive enhancement
- Lazy loading patterns
